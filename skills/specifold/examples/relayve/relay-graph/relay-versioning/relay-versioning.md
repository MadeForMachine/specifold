## Why

Relays are editable thoughts, but edits should not overwrite the history of what
was expressed. Versioning gives authored relays a simple append-only content
history while keeping publication and discovery pointed at the latest version.

## Contract

Provides: integer relay version numbers, immutable content snapshots, and the
rule that the latest version is the current public version. Consumes: relay
identity from the Relay graph and authored or Ghosty-regenerated content from
Studio/Ghosty.

## Essential vs accidental

Essential: versions count upward from 1, the latest version is the current public
version, and content changes create new versions. Versioned content includes the
relay body, TLDR, reader prompt, and style. TLDR and reader prompt are regenerated
whenever relay content is edited by the user or regenerated by Ghosty. References
are editable graph edges that pin the referenced parent version at link time and
can be removed by the user. Accidental: rollback tools, version comparison UI, or
manual editing of derived summaries.

## Rejected

Draft relays were rejected because a relay either exists or it does not. Pinning
publications to old versions was rejected for the MVP because stations should show
the latest relay without introducing editorial release management.
